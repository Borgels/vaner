## Suggested answer snippets (copy/paste)

### Security policy URL

`https://github.com/Borgels/vaner/blob/main/SECURITY.md`

### Vulnerability reporting process

Vaner accepts private vulnerability reports via `security@vaner.ai` and follows coordinated disclosure. We acknowledge reports within 2 days, complete triage within 7 days, and target public disclosure within 90 days unless a different window is agreed with the reporter.

### Development process

All changes go through GitHub pull requests against a protected `main` branch. CI, linting, and security checks run in GitHub Actions. Contributor and governance guidance are documented in `CONTRIBUTING.md`, `GOVERNANCE.md`, and `MAINTAINERS.md`.

### Release integrity

Vaner publishes signed release artifacts and provenance in GitHub Actions. Python release workflows generate attestations and SLSA provenance, and container releases are signed keylessly with cosign.
