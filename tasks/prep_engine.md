# Preparation Engine — Sequential Build Tasks
# Each line is one task for the builder agent.
# Run with: python work.py --plan tasks/prep_engine.md

# Task 1: Create the package skeleton
Create apps/vaner-daemon/src/vaner_daemon/preparation_engine/__init__.py (empty) and apps/vaner-daemon/src/vaner_daemon/preparation_engine/triggers.py. triggers.py should define a PrepTrigger dataclass with fields: context_key (str), active_files (list[str]), branch (str), reason (str, one of: file_changed|git_commit|branch_switch|manual), timestamp (float). Add a debouncer class Debouncer(min_interval_seconds: float) with a single method should_trigger(trigger: PrepTrigger) -> bool that returns False if a trigger with the same context_key was seen less than min_interval_seconds ago. Uses time.monotonic internally. Thread-safe. Write tests in apps/vaner-daemon/tests/test_triggers.py covering: debounce blocks rapid triggers, debounce allows after interval, different context_keys are independent. Run tests and lint before reporting done.

# Task 2: Artifact job planner
Create apps/vaner-daemon/src/vaner_daemon/preparation_engine/planner.py. Defines ArtifactJob dataclass (job_id: str, artifact_kind: str, source_path: str, context_key: str, priority: int) and PreparationPlanner class. PreparationPlanner.plan(trigger: PrepTrigger, existing_artifacts: list[Artefact]) -> list[ArtifactJob]: given a trigger and the current artifact store contents, return a list of ArtifactJobs to run. Rules: (1) for each active_file in trigger.active_files, if no fresh file_summary exists (check is_stale), add a file_summary job at HIGH priority; (2) if trigger.reason is git_commit or branch_switch, add a diff_summary job at CRITICAL priority for the repo root; (3) cap total jobs at 10 to avoid queue flooding; (4) skip files not ending in .py .ts .js .tsx .md .toml .yaml .yml. Import Artefact and is_stale from vaner_tools.artefact_store. Import Priority from vaner_runtime.job_queue. Write tests in apps/vaner-daemon/tests/test_planner.py. Run tests and lint before reporting done.

# Task 3: Artifact generator node
Create apps/vaner-daemon/src/vaner_daemon/preparation_engine/generator.py. Defines generate_file_summary(source_path: str, repo_root: Path, model_name: str = "qwen2.5-coder:32b") -> Artefact | None. Reads the file, calls ChatOllama with the standard file summary prompt (same as repo-analyzer: keyword-optimized, names classes/functions explicitly). Writes artefact to store via write_artefact. Returns Artefact on success, None on failure. Also define generate_diff_summary(repo_root: Path, model_name: str = "qwen2.5-coder:32b") -> Artefact | None — runs git diff HEAD --stat and git diff HEAD (truncated to 3000 chars), summarises what changed and what related code areas might be affected. Uses async def with await model.ainvoke(). Import from vaner_tools.artefact_store. Import ChatOllama from langchain_ollama. Write tests using pytest-asyncio and mock ChatOllama (mock ainvoke to return a fake response). Run tests and lint before reporting done.

# Task 4: LangGraph preparation workflow
Create apps/vaner-daemon/src/vaner_daemon/preparation_engine/graph.py. LangGraph StateGraph with these nodes in order: check_jobs -> [run_jobs or END] -> update_index -> END. State: PreparationState dataclass (trigger: PrepTrigger | None, jobs: list[ArtifactJob], completed: list[str], errors: list[str], job_id: str). check_jobs node: reads existing artifacts via list_artefacts, calls PreparationPlanner.plan(), returns {jobs: [...]}. If jobs is empty, routes to END. run_jobs node: iterates jobs, calls generate_file_summary or generate_diff_summary based on artifact_kind, appends to completed or errors. Batch max 3 concurrent with asyncio.gather. update_index node: re-reads all file_summary artifacts, rebuilds repo index JSON, writes to .vaner/cache/repo_index/root.json (same format as analyzer). Uses SqliteSaver from langgraph_checkpoint_sqlite for checkpointing. Graph is compiled with name="Vaner Preparation Engine". Add pyproject.toml dependency on langchain-ollama and langgraph-checkpoint-sqlite. Run tests and lint before reporting done.

# Task 5: Wire preparation engine into daemon
In apps/vaner-daemon/src/vaner_daemon/daemon.py, add PreparationEngine class that: holds a JobQueue from vaner_runtime, subscribes to state_engine.on_context_changed callback, runs the preparation LangGraph when triggered (respecting Debouncer with 5s interval), handles cancellation when state_engine.on_context_invalidated fires (cancels in-flight jobs by context_key via JobStore). Import PreparationPlanner, PrepTrigger, the preparation graph. Add _preparation_engine field to VanerDaemon, start it in VanerDaemon.start() after state engine, stop it in VanerDaemon.stop(). Update vaner daemon status to show: jobs_queued, jobs_running, last_prep_at, artifacts_in_cache. Write an integration test in apps/vaner-daemon/tests/test_daemon_integration.py that: starts a VanerDaemon on a temp git repo, touches a .py file, waits up to 30s for an artifact to appear in the cache. Run tests and lint before reporting done.
